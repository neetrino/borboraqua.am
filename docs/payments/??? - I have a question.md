# Ответ: настройки EHDM (шаг 3 плагина) и seq 200

## 1. Тип налогообложения (Հարկման տեսակը)

**В плагине (шаг 3):** выбор из 4 вариантов (ԱԱՀ-ով հարկվող / առանց ԱԱՀ / Շրջ. հարկ / Միկրո).

**У нас:** задаётся в ENV переменной **`EHDM_DEP`**:
- `1` = ԱԱՀ-ով հարկվող (с НДС) — как на скриншоте
- `2` = ԱԱՀ-ով չհարկվող (без НДС)
- `3` = Շրջ. հարկ (оборотный налог)
- `7` = Միկրոձեռնարկատիրություն (микро)

В `.env` у тебя уже стоит `EHDM_DEP=1` — то же, что «ԱԱՀ-ով հարկվող» в плагине.

---

## 2. Процент налогообложения (ԱԱՀ %)

**В плагине (шаг 3):** поле «ԱԱՀ (%)» = 16,67.

**У нас:** в API EHDM в запросе `/print` отдельного поля «процент НДС» нет — режим задаётся только через **dep**. Процент 16,67 в плагине использовался для отображения в тексте чека («Որից ԱԱՀ = …»). В нашем коде пока такого экрана нет. В `.env.example` добавлена **опциональная** переменная **`EHDM_VAT_PERCENT=16.67`** — её можно использовать, когда будем выводить текст чека/отчёта.

---

## 3. Галка «Связь с SKU продукта» (Ապրանքի SKU-ի համապատասխանեցնել ապրանքի կոդի հետ?)

**В плагине (шаг 3):** галочка включена — «код товара» в чеке = SKU товара.

**У нас:** так и сделано. В коде (`apps/web/lib/payments/ehdm/print.ts`) в теле запроса EHDM для каждой позиции передаётся **`goodCode: item.sku || "0"`**, т.е. код товара в чеке всегда берётся из SKU позиции заказа. Отдельной настройки «вкл/выкл связь с SKU» нет — поведение как при включённой галке в плагине.

---

## 4. EHDM_INITIAL_SEQ = 200

**Сделано:**
- В **`.env.example`** по умолчанию указано **`EHDM_INITIAL_SEQ="200"`** (раньше было 65), с комментарием, что 200+ нужно при переходе со старого сайта.
- В коде config по умолчанию используется **200**, если в ENV ничего не задано.
- В **документации** (`docs/payments/EHDM-INTEGRATION.md`) добавлены:
  - раздел **«6. Seq 200 — при переходе со старого site»** с объяснением;
  - рекомендация один раз выполнить в БД:  
    `UPDATE ehdm_state SET "nextSeq" = 200 WHERE id = 'default';`  
    если миграция уже создала строку с 65.

Так при запуске нового проекта seq не пересечётся со старым сайтом (где мог дойти до 65–100+).

---

Итог: тип налогообложения = `EHDM_DEP`, процент = опционально `EHDM_VAT_PERCENT`, связь с SKU у нас всегда как «галка включена», начальный seq по умолчанию = 200 и описан в доке.
